05Why this exercise mattered

A real bug the validation caught

Running real (non-flat) USDCNY rates exposed a genuine surface-interpolation bug: total variance was read at a single surface-wide forward while each smile was fitted in its own per-tenor forward. With USDCNY's forward running 7.26→6.54, long-tenor vols came out roughly doubled (5Y ATM: 10.5% vs 5.2%). Flat-rate unit tests couldn't see it. The fix stores per-tenor forwards; a regression test guards it. The same bug would have mispriced the P&L engine and risk grid under any real rate curve.

The modelling ladder

Garman-Kohlhagen + Greeks SVI surface + no-arb Dupire local vol Heston (char. fn) Local-stochastic vol 3-factor (FX + dual HW rates)

Each layer exists because the one below fails at something a trader pays for: BS can't see the smile; local vol gets the smile's motion wrong; Heston fixes motion but loses exact fit; LSV unifies both; the three-factor model un-freezes rates for long-dated exotics where FX–rates correlation is priced risk.
